📊 IMPROVED BATCH WORKFLOW
Based on water production facility best practices, here's an optimized workflow:
Phase 1: Pre-Production (Planning Office)
NO PRODUCTION DISRUPTION - All done before line start

✅ Batch Planning (Production Planner)

Create batch record
Assign components from inventory
Reserve materials (soft lock)
Set production date/shift


✅ Pre-Production QA Check (QA - Remote)

Verify component availability
Check material certificates
Approve batch plan
GATE 1 OPENS → Ready for production



Phase 2: Production Setup (15 min before start)
MINIMAL DISRUPTION - Quick checks only

✅ Water Treatment Verification (Operator)

Log RO conductivity
Log ozone residual
Upload filter status photos


✅ Line Setup Verification (Operator)

Scan bottle batch QR code
Scan cap batch QR code
Scan label batch QR code
System auto-fills material details


✅ Setup Approval (QA - Remote)

Review logged parameters
Approve via mobile device
GATE 2 OPENS → Start production



Phase 3: Production Run (No Stops)
ZERO DISRUPTION - Continuous operation

✅ Auto IPQC Logging (Operator - Every 30 min)

Tablet/mobile form appears automatically
Fill volume, cap torque, visual checks
Takes 60 seconds to complete
No production stop needed


✅ Real-time Monitoring (QA - Dashboard)

Live IPQC data stream
Auto-alerts if parameters drift
Can intervene remotely if needed



Phase 4: Post-Production (After line stop)
NO PRODUCTION DISRUPTION - Done after shift

✅ Yield Reconciliation (Operator)

Log final counts
System auto-calculates yield
Flag deviations


✅ Cleaning Verification (Operator)

Upload cleaning checklist photos
Log sanitization complete


✅ Final QA Release (QA Manager)

Review complete batch data
Digital signature release
GATE 3 OPENS → Batch released to warehouse
System auto-generates finished goods inventory




🏗️ DATABASE SCHEMA ADDITIONS
I'll show you the new tables needed alongside your existing inventory system:
sql-- Production Batches Table
CREATE TABLE production_batches (
    batch_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    batch_number VARCHAR(50) UNIQUE NOT NULL, -- e.g., FD-500-20250205-001
    batch_record_code VARCHAR(50) UNIQUE NOT NULL, -- QA-PRO-BAT-LOG-001
    product_id UUID REFERENCES products(product_id),
    production_date DATE NOT NULL,
    production_line VARCHAR(100),
    shift VARCHAR(20), -- 'day' or 'night'
    planned_quantity INTEGER NOT NULL,
    actual_output INTEGER,
    yield_percentage DECIMAL(5,2),
    
    -- Status tracking
    status VARCHAR(50) NOT NULL, -- 'planned', 'setup', 'in_progress', 'completed', 'released', 'rejected', 'on_hold'
    current_gate INTEGER DEFAULT 0, -- 0=planning, 1=pre-prod, 2=production, 3=complete
    
    -- Personnel
    line_supervisor_id UUID REFERENCES users(user_id),
    qa_approved_by UUID REFERENCES users(user_id),
    created_by UUID REFERENCES users(user_id),
    
    -- Timestamps
    created_at TIMESTAMP DEFAULT NOW(),
    setup_started_at TIMESTAMP,
    production_started_at TIMESTAMP,
    production_completed_at TIMESTAMP,
    qa_released_at TIMESTAMP,
    
    -- Metadata
    notes TEXT,
    rejection_reason TEXT
);

-- Batch Components (Links inventory to batches)
CREATE TABLE batch_components (
    component_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    batch_id UUID REFERENCES production_batches(batch_id),
    inventory_id UUID REFERENCES inventory(inventory_id),
    component_type VARCHAR(50), -- 'bottle', 'cap', 'label', 'water'
    
    -- Quantity tracking
    planned_quantity INTEGER NOT NULL,
    actual_consumed INTEGER,
    wastage INTEGER DEFAULT 0,
    
    -- Material details
    supplier_batch_lot VARCHAR(100),
    material_status VARCHAR(20) DEFAULT 'reserved', -- 'reserved', 'consumed', 'returned'
    
    assigned_at TIMESTAMP DEFAULT NOW(),
    consumed_at TIMESTAMP
);

-- QA Gates (Checkpoint tracking)
CREATE TABLE batch_qa_gates (
    gate_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    batch_id UUID REFERENCES production_batches(batch_id),
    gate_number INTEGER NOT NULL, -- 1, 2, 3
    gate_name VARCHAR(100), -- 'Pre-Production Check', 'Line Setup Approval', 'Final Release'
    
    status VARCHAR(20) DEFAULT 'pending', -- 'pending', 'approved', 'rejected'
    required_by_role VARCHAR(50), -- 'qa', 'qa_manager', 'supervisor'
    
    approved_by UUID REFERENCES users(user_id),
    approved_at TIMESTAMP,
    rejection_reason TEXT,
    
    -- Checklist data (JSON for flexibility)
    checklist_data JSONB,
    
    created_at TIMESTAMP DEFAULT NOW()
);

-- IPQC Records (In-Process Quality Control)
CREATE TABLE batch_ipqc_records (
    ipqc_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    batch_id UUID REFERENCES production_batches(batch_id),
    
    check_time TIMESTAMP NOT NULL,
    checked_by UUID REFERENCES users(user_id),
    
    -- Measurements
    fill_volume_ml DECIMAL(6,2),
    cap_torque_nm DECIMAL(4,2),
    visual_inspection BOOLEAN,
    label_position_ok BOOLEAN,
    coding_legibility_ok BOOLEAN,
    
    -- Status
    all_checks_passed BOOLEAN,
    notes TEXT,
    
    created_at TIMESTAMP DEFAULT NOW()
);

-- Water Treatment Logs
CREATE TABLE batch_water_treatment_logs (
    log_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    batch_id UUID REFERENCES production_batches(batch_id),
    
    -- Parameters
    ro_conductivity_us_cm DECIMAL(6,2), -- μS/cm
    ozone_residual_ppm DECIMAL(4,2),
    sand_filter_status BOOLEAN,
    carbon_filter_status BOOLEAN,
    uv_system_status BOOLEAN,
    ozone_injection_active BOOLEAN,
    
    verified_by UUID REFERENCES users(user_id),
    logged_at TIMESTAMP DEFAULT NOW()
);

-- Line Setup Parameters
CREATE TABLE batch_line_setup (
    setup_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    batch_id UUID REFERENCES production_batches(batch_id),
    
    -- Equipment parameters
    rinsing_pressure_mpa DECIMAL(4,2),
    fill_volume_target_ml DECIMAL(6,2),
    fill_volume_actual_ml DECIMAL(6,2),
    cap_torque_target_nm DECIMAL(4,2),
    cap_torque_actual_nm DECIMAL(4,2),
    line_speed_bph INTEGER, -- Bottles per hour
    
    verified_by UUID REFERENCES users(user_id),
    setup_at TIMESTAMP DEFAULT NOW()
);

-- Batch Coding & Traceability
CREATE TABLE batch_coding (
    coding_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    batch_id UUID REFERENCES production_batches(batch_id),
    
    batch_code_format VARCHAR(100),
    batch_code_printed VARCHAR(100),
    expiry_date DATE,
    best_before_date DATE,
    coding_verified BOOLEAN,
    
    verified_by UUID REFERENCES users(user_id),
    verified_at TIMESTAMP DEFAULT NOW()
);

-- Post-Production Cleaning
CREATE TABLE batch_cleaning_logs (
    cleaning_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    batch_id UUID REFERENCES production_batches(batch_id),
    
    line_flush_completed BOOLEAN DEFAULT FALSE,
    equipment_cleaning_completed BOOLEAN DEFAULT FALSE,
    area_sanitation_completed BOOLEAN DEFAULT FALSE,
    
    -- Photo evidence
    photo_urls TEXT[], -- Array of URLs to uploaded photos
    
    completed_by UUID REFERENCES users(user_id),
    completed_at TIMESTAMP DEFAULT NOW()
);

-- Deviations/Incidents
CREATE TABLE batch_deviations (
    deviation_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    batch_id UUID REFERENCES production_batches(batch_id),
    
    incident_time TIMESTAMP NOT NULL,
    description TEXT NOT NULL,
    action_taken TEXT,
    ncr_number VARCHAR(50), -- Non-Conformance Report reference
    
    reported_by UUID REFERENCES users(user_id),
    created_at TIMESTAMP DEFAULT NOW()
);

-- Indexes for performance
CREATE INDEX idx_batches_status ON production_batches(status);
CREATE INDEX idx_batches_date ON production_batches(production_date);
CREATE INDEX idx_batches_product ON production_batches(product_id);
CREATE INDEX idx_components_batch ON batch_components(batch_id);
CREATE INDEX idx_components_inventory ON batch_components(inventory_id);
CREATE INDEX idx_qa_gates_batch ON batch_qa_gates(batch_id);
CREATE INDEX idx_ipqc_batch ON batch_ipqc_records(batch_id);
```

🔄 WORKFLOW AUTOMATION
Auto-Triggers & Notifications
javascript// Pseudo-code for automated workflows

// 1. When batch is submitted for approval
ON batch.status = 'awaiting_qa_approval':
    NOTIFY qa_team via email & mobile
    CREATE qa_gate record (gate_number=1)
    RESERVE inventory components (soft lock)

// 2. When QA approves Gate 1
ON qa_gate.gate_number = 1 AND status = 'approved':
    UPDATE batch.status = 'ready_for_setup'
    UPDATE batch.current_gate = 1
    NOTIFY line_supervisor via mobile
    HARD LOCK inventory components

// 3. When operator starts setup
ON batch.status = 'setup':
    START timer for setup duration
    ENABLE water treatment logging
    ENABLE material scanning

// 4. When all setup data logged
ON batch has all setup_data:
    CREATE qa_gate record (gate_number=2)
    NOTIFY qa_team for approval

// 5. When QA approves Gate 2
ON qa_gate.gate_number = 2 AND status = 'approved':
    UPDATE batch.status = 'in_progress'
    UPDATE batch.current_gate = 2
    UPDATE batch.production_started_at = NOW()
    START IPQC timer (every 30 min)

// 6. IPQC Auto-Reminders
EVERY 30 minutes WHILE batch.status = 'in_progress':
    SEND mobile notification to operator
    IF missed 2 checks:
        ESCALATE to supervisor
        SEND alert to QA

// 7. IPQC Auto-Analysis
ON new ipqc_record:
    IF fill_volume OUT OF RANGE:
        SEND immediate alert to QA & supervisor
        FLAG batch for review
    IF 3 consecutive failures:
        AUTO PAUSE batch
        REQUIRE supervisor override

// 8. When operator completes production
ON batch.status = 'completed':
    AUTO CALCULATE yield
    DEDUCT consumed materials from inventory
    CREATE finished goods inventory entry
    CREATE qa_gate record (gate_number=3)
    NOTIFY qa_manager for final release

// 9. When QA releases batch
ON qa_gate.gate_number = 3 AND status = 'approved':
    UPDATE batch.status = 'released'
    UPDATE inventory.available_quantity (finished goods)
    AUTO GENERATE batch report PDF
    EMAIL batch report to stakeholders
    ARCHIVE batch record
```
